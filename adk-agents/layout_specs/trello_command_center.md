# Trello Command Center Layout Specification

> **Aesthetic**: Raw Data Interface + Full Control Panel
> **Purpose**: Complete Trello management - Extract, Create, Modify, Delete, Sync

## Design Philosophy

### Core Principle: Full API Coverage
This interface exposes **ALL** Trello API capabilities:

| Category | Operations |
|----------|------------|
| **READ** | Boards, Lists, Cards, Checklists, Custom Fields, Actions, Attachments |
| **CREATE** | Cards, Lists, Checklists, Comments, Labels, Attachments |
| **UPDATE** | Edit cards, Move between lists, Archive/Unarchive, Mark checklist items |
| **DELETE** | Archive cards, Remove attachments, Delete comments |
| **SEARCH** | Advanced search with operators (@member, #label, due:week, has:attachments) |
| **SYNC** | Webhooks status, Real-time updates, Batch operations |

### UX Principles
- **Zero friction** - every action is 1-2 clicks away
- **Copy-friendly** - all data easily selectable/copiable
- **Keyboard-first** - power users can work without mouse
- **Dense but readable** - maximize information per screen
- **Non-destructive defaults** - archive instead of delete, confirm dangerous actions

## Visual Style

### Inspiration
- Terminal/CLI interfaces (clarity, density)
- VS Code's Output panel (raw, scannable)
- DataGrip/TablePlus (database management)
- Linear (modern task management)

## API Mapping

| UI Action | API Endpoint | Method |
|-----------|-------------|--------|
| Load boards | `/1/members/me/boards` | GET |
| Load lists | `/1/boards/{id}/lists` | GET |
| Load cards | `/1/boards/{id}/cards?customFields=true` | GET |
| Search | `/1/search?query={q}&modelTypes=cards` | GET |
| Create card | `/1/cards` | POST |
| Update card | `/1/cards/{id}` | PUT |
| Move card | `/1/cards/{id}` (idList) | PUT |
| Archive card | `/1/cards/{id}` (closed=true) | PUT |
| Delete card | `/1/cards/{id}` | DELETE |
| Add comment | `/1/cards/{id}/actions/comments` | POST |
| Add attachment | `/1/cards/{id}/attachments` | POST |
| Update checklist item | `/1/cards/{id}/checkItem/{itemId}` | PUT |
| Set custom field | `/1/cards/{id}/customField/{fieldId}/item` | PUT |

## Search Operators Reference

| Operator | Example | Description |
|----------|---------|-------------|
| `@member` | `@me`, `@pedro` | Cards assigned to member |
| `#label` | `#urgente` | Cards with label |
| `due:` | `due:day`, `due:week` | Cards due in timeframe |
| `created:` | `created:14` | Cards created in last N days |
| `has:` | `has:attachments` | Cards with attachments |
| `is:` | `is:archived` | Card status |
| `-` | `-has:members` | Negative (cards WITHOUT) |
| `board:` | `board:casos` | Limit to board |
| `list:` | `list:fazendo` | Limit to list |

## Performance Requirements

- Load 100+ cards in <500ms
- Instant filtering (client-side)
- Optimistic UI updates
- Background sync every 30s
- Debounced search (300ms)
- Batch operations for 10+ cards

## Accessibility

- WCAG AAA contrast (7:1 minimum)
- Full keyboard navigation
- Screen reader announcements for:
  - Selection changes
  - Operation results
  - Error states
- Focus visible on all interactive elements
- Reduced motion option
